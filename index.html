每日LC之摩尔计数法求众数

Leetcode 169 多数元素 – 求众数

法1： 摩尔计数法：时间复杂度O(n), 空间复杂度O(1)

遍历数组：

  • 当前数是众数，次数加一
  • 当前数不是众数，次数减一：
    • 如果次数 <= 0， 众数变为当前数，次数重置为1

遍历结束，返回众数

法2： 排序后返回arr[len/2]即可

时间复杂度O(nlogn),空间复杂度O(1)

每日LC之环形链表入口

Leetcode 142 环形链表Ⅱ – 判断链表是否有环，入口在哪

一、基础：判断有环

快慢指针，快指针一次走两步，慢指针一次一步。如果某次他们指向同一个节点，说明相遇，说明有环

注意：限制条件：while(fast!=null && slow!=null && fast.next!=null)

开始节点：我采用了dummy节点，fast和slow都从此开始

Leetcode 141 环状链表

二、进阶：判断入口

推公式：

  • 假设：到换入口前无环的距离为z

  • 同时开始，慢指针走完z，快指针走过了2z， 用环内距离表示就是2z = z + k*(x+y) + x, k为环内走的圈数，x为最新一圈走过的环内距离, x+y = 环的周长

  • 等式： 2z = z + k*(x+y) + x， –> z = k*(x+y) + x

  • 环内追及： t*2 = t*1 + y –> y = t 慢指针领先y，因为慢指针入环时快指针该圈走了x; 快慢追及上时慢指针走了t*1 = y*1 = y, 距离下一圈x

  • 此时一个新指针从dummy出发,速度为1，走到环口t = z = k*(x+y) + x;

    慢指针这段时间内可以走： k * (x+y) + x, 同时它本身就可以走 k2 * (x+y) + x，所以k == k2

    so, 他们在环口相遇

  • 结论：快慢指针相遇时，一个新指针从dummy出发，新指针和慢指针相遇的节点即为环入口

1. 前缀树

前缀树是一种N叉树，每一个节点代表一个字符串或前缀

注意：根节点为空字符串

2. 子节点如何表示

1. 数组 SubNode[26] nodes: targetChar - 'a'为下标

查询时看当前节点的子节点数组是否有相应字符 SubNode[thisChar - 'a']是否为空

插入时先查询，如果有则进入该节点；若无则插入

2. HashMap： key = 字符， value = SubNode

在只考虑26个字母的情况下，没有比数组查询慢很多，但不用预先创建长度为26TreeNode的数组
